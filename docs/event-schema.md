# Event Schema Reference

Every Sandtrace audit event follows the [AuditEvent schema](../schema/event.json). Events are written as JSONL (one JSON object per line).

## Event structure

```json
{
  "schema_version": "1.0",
  "event_id": "evt_a1b2c3",
  "event_type": "network_egress",
  "agent_id": "agent-xyz",
  "trace_id": "trace-001",
  "seq": 1,
  "prev_hash": null,
  "record_hash": "sha256:abcdef...",
  "wall_time": "2025-03-25T12:00:00Z",
  "evidence_tier": "hypervisor",
  "payload": { ... },
  "verdict": { ... }
}
```

## Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `schema_version` | string | yes | Always `"1.0"` |
| `event_id` | string | yes | Unique event identifier (UUID) |
| `event_type` | string | yes | One of the [event types](#event-types) below |
| `agent_id` | string | yes | Identifier of the agent being traced |
| `trace_id` | string | yes | Groups events from the same trace session |
| `seq` | integer | yes | Sequence number, starting at 1, strictly increasing |
| `prev_hash` | string or null | no | Hash of the previous event's `record_hash`. `null` for the first event |
| `record_hash` | string | yes | SHA-256 hash of this event's content plus `prev_hash`. See [hash chain](verifying-audit-trails.md) |
| `wall_time` | string | yes | RFC 3339 timestamp of when the event was captured |
| `evidence_tier` | string | yes | One of the [evidence tiers](#evidence-tiers) below |
| `payload` | object | yes | Event-type-specific data |
| `verdict` | object | no | Policy evaluation result, if a policy was loaded |

## Event types

### `network_egress`

Captured from AF_PACKET socket on the Firecracker tap device. Records every TCP/UDP connection from the guest VM to an external destination.

**Evidence tier:** `hypervisor`

**Payload fields:**

| Field | Type | Description |
|-------|------|-------------|
| `dest_host` | string | Resolved hostname or IP address |
| `dest_port` | u16 | TCP/UDP destination port |
| `bytes_sent` | u64 | Bytes sent from VM to destination |
| `bytes_received` | u64 | Bytes received from destination |
| `packet_count` | u64 | Number of packets in the connection |
| `protocol` | string | `"tcp"` or `"udp"` |
| `description` | string | (optional) Application-layer context |

**Example:**

```json
{
  "event_type": "network_egress",
  "evidence_tier": "hypervisor",
  "payload": {
    "dest_host": "api.stripe.com",
    "dest_port": 443,
    "bytes_sent": 316,
    "bytes_received": 1024,
    "packet_count": 8,
    "protocol": "tcp"
  }
}
```

### `filesystem_summary`

Captured from OverlayFS upper-dir monitoring or block-device snapshot diffs. Records filesystem changes made by the guest.

**Evidence tier:** `hypervisor`

**Payload fields:**

| Field | Type | Description |
|-------|------|-------------|
| `files_created` | string[] | Paths of files created |
| `files_modified` | string[] | Paths of files modified |
| `files_deleted` | string[] | Paths of files removed (overlay whiteouts) |
| `total_bytes_written` | u64 | Sum of all written file sizes |
| `file_sizes` | object | (optional) Map of `{path: bytes}` |
| `path` | string | (optional) Single file path being accessed |

**Example:**

```json
{
  "event_type": "filesystem_summary",
  "evidence_tier": "hypervisor",
  "payload": {
    "files_created": ["/home/agent/.agent_memory.json"],
    "files_modified": ["/tmp/output.txt"],
    "files_deleted": [],
    "total_bytes_written": 2048,
    "file_sizes": {
      "/home/agent/.agent_memory.json": 512,
      "/tmp/output.txt": 1536
    }
  }
}
```

### `syscall_activity`

Captured via ptrace on the Firecracker jailer process. Records system call activity from the guest.

**Evidence tier:** `kernel`

**Payload fields:** Implementation-specific. Contains syscall name and argument details.

### `policy_violation`

Generated by the policy evaluator when an event receives a `deny` or `anomaly` verdict. Contains a copy of the violating event's payload plus verdict details.

**Evidence tier:** `hypervisor`

## Evidence tiers

Evidence tiers indicate the trust level of the capture mechanism:

| Tier | Mechanism | Trust basis |
|------|-----------|-------------|
| `hypervisor` | AF_PACKET tap capture, OverlayFS monitoring, block snapshots | Host-side, outside the guest — agent cannot influence instrumentation |
| `kernel` | ptrace syscall hooks on the jailer process | Kernel-level interposition |

The `hypervisor` tier is the strongest: the agent runs inside the guest VM and has no access to the host-side capture infrastructure.

## Verdict object

When a policy is loaded, each event includes a `verdict`:

```json
{
  "verdict": {
    "result": "allow",
    "policy_rule": "tool:stripe_charge",
    "reason": "matched rule tool:stripe_charge for api.stripe.com:443"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `result` | string | `"allow"`, `"deny"`, or `"anomaly"` |
| `policy_rule` | string | ID of the matched rule, or `"none"` if no rule matched |
| `reason` | string | Human-readable explanation |

The verdict is **not** included in the `record_hash` computation, allowing policy re-evaluation without breaking the hash chain.
